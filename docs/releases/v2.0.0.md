# v2.0.0 — Lightweight tracks + scope routing

First v2 release. Introduces three lighter tracks alongside `/pipeline`
and `/hotfix`, plus scope-based routing so small changes don't pay the
full nine-stage tax.

## Highlights

- **`/quick`** — single-area, ≤ ~100 LOC. Skips design; one cross-area
  reviewer instead of the 3×2 matrix.
- **`/config-only`** — 100% config diff. Lint + no-regression tests +
  optional deploy.
- **`/dep-update`** — package upgrades. Changelog scan + SCA review.
- **Safety stoplist** — auth/crypto/PII/payments/migrations/new-deps/new-FF
  always take full `/pipeline`.
- Every gate file now carries `"track": "<name>"` so downstream tooling
  can branch on track.

## Breaking

- **Gate files** gain a `"track"` field. Strict-schema parsers should
  relax to ignore unknown fields.
- **`/pipeline` prompts** for track choice when a request looks like a
  fit for a lighter track. Prefix requests with `TRACK: full —` to skip
  the prompt in scripted flows.
