# v2.3.1 — Scoped peer review + approval-derivation hook

Closes the "any agent can approve anyone" hole from v1/v2 and adds a
scoped review shape for area-contained diffs.

## Highlights

### Approval-derivation hook

New `.claude/hooks/approval-derivation.js` runs as PostToolUse on
Write/Edit, parses per-area sections in
`pipeline/code-review/by-<reviewer>.md` for `REVIEW: APPROVED` /
`REVIEW: CHANGES REQUESTED` markers, and reconciles
`pipeline/gates/stage-05-<area>.json`. The hook is now the **single
writer** on Stage 5 gate approvals — agents can no longer author their
own entries.

### Scoped peer review

When the diff is area-contained, Stage 5 uses `review_shape: "scoped"`
and `required_approvals: 1` (one cross-area reviewer). Cross-area
diffs still use the v1 matrix with `required_approvals: 2`.

### Review file format

```markdown
# Review by dev-frontend

## Review of backend
<comments>
REVIEW: APPROVED

## Review of platform
<comments>
REVIEW: CHANGES REQUESTED
BLOCKER: <text>
```

## Breaking

- **Review file format changed.** Legacy single-verdict review files
  (no `## Review of <area>` header) are ignored by the hook — the
  gate stays at `FAIL` until a properly-formatted review arrives.
- **Agents must not author `approvals` on stage-05 gates** directly.
  Any direct write is overwritten on the next reviewer file save. The
  hook is authoritative.
- **Stage 5 gate `changes_requested` shape** changed from `[string]`
  to `[{reviewer, timestamp}]`. Read `entry.reviewer` going forward.

Mid-flight Stage 5 reviews should either finish under the v2.3 format
or `/reset` and restart under v2.3.1.
