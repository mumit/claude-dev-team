# v2.6.0 — Automation layer

Adds a Node CLI (`scripts/claude-team.js`), 15 helper scripts, JSON schemas for
every pipeline gate, 11 pipeline artifact templates, an `examples/tiny-app`
dogfood project, and a dedicated `reviewer` agent. Test suite grows from 30 to
265 tests. No slash-command behaviour changed; no breaking changes.

## Highlights

### `scripts/claude-team.js` — Node CLI

Mirrors all 23 slash commands as subcommands and adds automation-only helpers:

```bash
node scripts/claude-team.js help          # full command list
node scripts/claude-team.js doctor        # verify framework files present
node scripts/claude-team.js status        # gate summary + run state
node scripts/claude-team.js validate      # validate gates against schemas
node scripts/claude-team.js next          # what stage runs next
node scripts/claude-team.js autofold      # fold Stage 7 when criteria are 1:1
```

All commands are also available via `npm run <command>` shims. Slash commands
remain authoritative inside an active Claude Code session; the CLI is the
integration layer for CI and non-Claude environments.

### JSON schemas (`schemas/`)

`gate.schema.json` plus `stage-01` through `stage-09` (including `stage-04a`
for the Stage 4.5a pre-review gate). Used by `npm run validate` and the test
suite to guard against schema drift.

### Templates (`templates/`)

11 scaffold stubs that `node scripts/claude-team.js pipeline:scaffold` seeds
into a new pipeline run. Thin by design — agents fill them out. Note: these are
distinct from `docs/*-template.md`, which are detailed reference documents with
explanatory prose used as agent guidance. Both sets are canonical and complement
each other.

### `reviewer` agent

A dedicated Stage 5 peer-reviewer agent (`sonnet` model; READ-ONLY; writes only
to `pipeline/code-review/by-<role>.md`). The `approval-derivation.js` hook
derives stage-05 gates from review files automatically — reviewer never writes
gates directly. Formalises and scopes what was previously an embedded review
pass inside dev-* agents. The team is now eight agents.

### Test suite (30 → 265 tests)

New suites: gate schema validation, helper-script unit tests, CLI subcommand
tests (`claude-team.js`), parity check, status readiness, release helper, and
presentation builder. All 265 pass with no external test framework.

## Breaking

None. Gate schemas, adapter contracts, agent file-permission scopes, and slash-
command behaviour are unchanged. The reviewer agent is additive.

## Migration

No migration steps required. Re-run `bootstrap.sh` (or `node scripts/bootstrap.js`)
to pick up the updated agent files including `reviewer.md`.
