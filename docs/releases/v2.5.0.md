# v2.5.0 — Budget + learning loop (v2.x stack complete)

Closes out the v2.x line with four additions, three of which are
opt-in so they don't disturb projects that don't want the overhead.

## Highlights

### Budget gate (opt-in, off by default)

Token + wall-clock budget tracked per run in `pipeline/budget.md`. On
exceed: `escalate` halts, `warn` logs and continues. Guardrail, not a
hard limit.

### PATTERN positive review tag

Reviewers can flag things done especially well using `PATTERN:` lines
inside Stage 5 review sections. Principal harvests PATTERN entries
during Step 9b synthesis and can promote recurring ones as positive
rules in `lessons-learned.md` ("Use X because …") rather than
corrective rules ("Don't do Y because …").

### Lesson auto age-out

Rules in `lessons-learned.md` not reinforced in 10 runs with current
`Reinforced: 0` retire automatically during synthesis. Distinguished
from explicit retirement in the new `aged_out` gate field.

### Async-friendly checkpoints (opt-in per checkpoint)

`checkpoints.{a,b,c}.auto_pass_when` config. Supported conditions:
`no_warnings`, `all_criteria_passed` (Checkpoint C only). Default
behaviour unchanged (wait for human). Security-sensitive work is
unaffected — the safety stoplist and Stage 4.5b security-engineer
veto remain authoritative.

## Breaking

- **Stage 9 gate schema extended** (additive): `aged_out`,
  `patterns_harvested`. `contributions_written` can now have 6
  entries when `security-engineer` participated in the retro —
  tooling that hardcoded the five-agent list must update.
- **Long-lived `lessons-learned.md` files** may shrink on the first
  v2.5 retros from age-out. Audit trail in retro synthesis blocks.
