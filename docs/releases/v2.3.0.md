# v2.3.0 — dev-qa + security-engineer + Stage 4.5

Separates three judgement calls that were all on `dev-platform` in
v1–v2.2: test authoring, security review, and infra/deploy. Each now
has its own seat with its own accountability.

## Highlights

### New agents

- **`dev-qa` (sonnet)** — owns `src/tests/` and Stage 6 test execution.
  Authors the 1:1 criterion-to-test mapping that drives v2.2's Stage 7
  auto-fold. Peer-review focus: testability.
- **`security-engineer` (opus)** — promoted from the `security-checklist`
  skill. Invoked on Stage 4.5b when the triggering heuristic matches
  (auth/crypto/PII/payments paths, dependency changes, Dockerfile /
  IaC changes, new env vars). Has veto power — a `veto: true` gate
  halts the pipeline regardless of peer-review approvals.

### New pipeline stages

- **Stage 4.5a** — `dev-platform` runs lint + type-check + SCA +
  license allowlist. Always runs on full track. Reviewers don't spend
  tokens on what the toolchain already flags.
- **Stage 4.5b** — `security-engineer` review. Runs only when the
  heuristic fires; skip decision recorded in `pipeline/context.md`
  under `## Brief Changes` as `SECURITY-SKIP: <reason>`.

Both must pass before Stage 5. `/hotfix` skips 4.5a when the
blast-radius constraint is tight; it does **not** skip 4.5b when the
heuristic fires.

### Stage 6 field addition

`criterion_to_test_mapping_is_one_to_one` — `dev-qa` sets it; it gates
the v2.2 Stage 7 auto-fold.

## Breaking

- **Scripts invoking `dev-platform`** directly for `/stage 6` or
  test-related work must switch to `dev-qa`. `/pipeline` handles
  routing automatically.
- **Downstream tooling enumerating gates** must accept
  `stage-04-pre-review` and `stage-04-security` as valid stages.
- **Legacy Stage 6 gates** lack `criterion_to_test_mapping_is_one_to_one`.
  Parsers should default missing → `false` (safer: triggers manual
  PM sign-off at Stage 7).
- **`security-checklist` skill demoted** from primary security rubric
  to `security-engineer`'s review frame. Other agents should not lean
  on it as a proxy for security review.
