# v2.1.0 — Gate validator hardening

Closes four enforcement gaps the v1 validator didn't catch. Each was
documented in prose since v1 but not checked in code.

## Highlights

- **Bypassed-escalation sweep** — validator checks all gates, not just
  the most recent. An older `ESCALATE` with a newer gate after it
  exits 3 with `BYPASSED ESCALATION`.
- **Retry integrity** — gates with `retry_number >= 1` must carry a
  non-empty `this_attempt_differs_by` or exit 1.
- **Track-field advisory** — non-blocking warning when v2.0's
  `"track"` field is missing or unrecognised.
- **Lessons-learned format validation** — malformed `**Reinforced:**`
  lines are reported with line numbers. Valid forms:
  `**Reinforced:** 0` or `**Reinforced:** <N> (last: YYYY-MM-DD)`.

## Breaking

- **Retry gates with empty `this_attempt_differs_by` now fail** —
  previously silent pass. State the delta explicitly when re-writing
  a retry gate.
- **Unresolved `ESCALATE` gates from prior runs halt new runs.** Run
  `/reset` before the first v2.1 pipeline, or manually resolve by
  writing a follow-up gate of any status.
